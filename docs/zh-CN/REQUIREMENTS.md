# 需求分析

## 1. 问题陈述

大语言模型（LLM）需要一种标准化的方式来与计算机交互——查看屏幕、控制鼠标和键盘、管理窗口——以代表用户执行现实世界的任务。模型上下文协议（MCP）为工具调用提供了标准化的传输层，但需要具体的服务器实现。

---

## 2. 功能需求

### FR1: 屏幕截图

| # | 描述 |
|---|-----|
| **FR1.1** | 捕获全屏（所有显示器合并或指定显示器） |
| **FR1.2** | 捕获由 (x, y, width, height) 指定的矩形区域 |
| **FR1.3** | 将截图保存到可配置目录（默认：系统临时目录） |
| **FR1.4** | 返回结构化数据：文件路径、时间戳、尺寸、坐标 |

### FR2: 窗口管理

| # | 描述 |
|---|-----|
| **FR2.1** | 列出所有可见窗口及其标题和几何信息 |
| **FR2.2** | 按部分标题匹配查找窗口（不区分大小写） |
| **FR2.3** | 通过标题聚焦（置顶）窗口 |
| **FR2.4** | 截取指定窗口的截图（聚焦 + 截图） |
| **FR2.5** | 返回窗口几何信息（位置、尺寸）及截图路径 |

### FR3: 鼠标控制

| # | 描述 |
|---|-----|
| **FR3.1** | 在指定坐标点击（单击、双击、多次点击） |
| **FR3.2** | 在指定坐标长按指定时长 |
| **FR3.3** | 从一个坐标拖拽到另一个坐标 |
| **FR3.4** | 移动光标但不点击 |
| **FR3.5** | 获取当前光标位置 |
| **FR3.6** | 滚轮操作（上/下滚动，可选位置） |
| **FR3.7** | 支持左键、右键、中键 |

### FR4: 键盘控制

| # | 描述 |
|---|-----|
| **FR4.1** | 按下单个键或快捷键组合（如 Ctrl+C） |
| **FR4.2** | 按住键指定时长 |
| **FR4.3** | 逐字符输入文本 |
| **FR4.4** | 执行带延迟的键盘操作定时序列 |

### FR5: 组合操作

| # | 描述 |
|---|-----|
| **FR5.1** | 在一次调用中执行混合鼠标 + 键盘操作序列 |
| **FR5.2** | 在序列中支持内联等待和截图 |
| **FR5.3** | 每个步骤支持可选的操作后延迟 |

### FR6: 附加操作

| # | 描述 |
|---|-----|
| **FR6.1** | 获取/设置剪贴板文本 |
| **FR6.2** | 通过命令/路径启动应用程序 |
| **FR6.3** | 在默认浏览器中打开 URL |
| **FR6.4** | 暂停执行 N 秒 |
| **FR6.5** | 获取指定坐标的像素颜色 |
| **FR6.6** | 按下快捷键组合（便捷封装） |

### FR7: 文档

| # | 描述 |
|---|-----|

## 3. 非功能需求

### NFR1: 跨平台

必须在 Windows、macOS 和 Linux 上运行（至少支持 Windows）。

### NFR2: 性能

- 截图捕获 < 500ms
- 鼠标/键盘操作 < 100ms 延迟

### NFR3: 可靠性

- 优雅的错误处理，返回结构化错误响应
- 窗口不存在或坐标无效时不崩溃

### NFR4: 可扩展性

- 无需修改服务器核心即可轻松添加新工具
- 工具逻辑与 MCP 协议清晰分离

### NFR5: 可维护性

- 清晰的文件结构和明确的模块边界
- 全程使用类型提示
- 所有公共函数都有文档字符串

---

## 4. 约束条件

- Python 3.10+
- 必须使用 MCP Python SDK（`mcp` 包）
- 截图存储在本地文件系统中（不在 MCP 响应中以二进制数据返回）
- 鼠标/键盘自动化通过 `pyautogui` 实现
- 屏幕截图通过 `mss` + `Pillow` 实现
