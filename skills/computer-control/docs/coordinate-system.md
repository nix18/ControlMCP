# 坐标系统参考

## 核心概念

ControlMCP 使用 **屏幕绝对坐标系**。所有 `mouse_click`、`mouse_move`、`mouse_drag` 的参数都是屏幕坐标。

## 坐标系示意

```
单显示器:
(0,0)┌─────────────────────────┐
     │  主显示器               │
     │  1920 × 1080            │
     │                         │
     │  (960, 540) = 屏幕中心  │
     │                         │
     │                    (1919, 1079)
     └─────────────────────────┘

双显示器（左+右）:
(0,0)┌──────────────┐(1920,0)┌──────────────────┐
     │  显示器1     │        │  显示器2         │
     │  1920×1080   │        │  2560×1440       │
     │              │        │                  │
     └──────────────┘        │            (4479, 1439)
                             └──────────────────┘

双显示器（上+下 或 左+右 有偏移）:
取决于系统设置，用 get_screen_info() 确认。
```

## 从截图到点击的完整流程

### 情况 1：使用 capture_window

```
capture_window 返回:
{
  "window_x": 286,        ← 窗口左上角的屏幕 X
  "window_y": 22,         ← 窗口左上角的屏幕 Y
  "window_width": 1404,   ← 窗口宽度
  "window_height": 990,   ← 窗口高度
  "screenshot_width": 1404,
  "screenshot_height": 990
}

截图中你看到了一个按钮在像素位置 (120, 45)

屏幕坐标 = (286 + 120, 22 + 45) = (406, 67)
```

### 情况 2：使用 capture_screen（全屏截图）

```
capture_screen 返回:
{
  "width": 1920,
  "height": 1080,
  "x": 0,
  "y": 0
}

截图中的按钮位置 (960, 540) 就是屏幕坐标，无需转换。
```

### 情况 3：使用 capture_region（区域截图）

```
capture_region(x=100, y=200, width=400, height=300) 返回:
{
  "x": 100,   ← 截图区域左上角的屏幕 X
  "y": 200,   ← 截图区域左上角的屏幕 Y
}

截图中按钮在 (50, 30)

屏幕坐标 = (100 + 50, 200 + 30) = (150, 230)
```

## 通用转换公式

```
screen_x = 截图左上角的屏幕X + 按钮在截图中的X
screen_y = 截图左上角的屏幕Y + 按钮在截图中的Y
```

| 截图工具 | 截图左上角来源 |
|---------|-------------|
| `capture_window` | `window_x`, `window_y` |
| `capture_screen` | `x`, `y`（通常为 0,0） |
| `capture_region` | 传入的 `x`, `y` 参数 |

## 常见 UI 元素尺寸参考

用于估计点击精度要求：

| 元素类型 | 典型尺寸 | 点击难度 |
|---------|---------|---------|
| 大按钮（如"提交"） | 120×40 px | 容易 |
| 普通按钮 | 80×30 px | 较容易 |
| 工具栏图标 | 16-24×16-24 px | 困难（建议用快捷键） |
| 菜单项 | 宽度可变×24 px | 中等 |
| 标签页 | 150×28 px | 较容易 |
| 下拉选项 | 宽度可变×20 px | 中等 |
| 复选框 | 16×16 px | 困难 |
| 单选按钮 | 16×16 px | 困难 |
| 输入框 | 可变×28-36 px | 容易（点中心） |

## DPI 缩放处理

Windows 高 DPI 缩放会影响坐标。

```
显示器物理分辨率: 3840 × 2160 (4K)
Windows 缩放: 150%
逻辑分辨率: 3840/1.5 × 2160/1.5 = 2560 × 1440

get_screen_info() 返回逻辑分辨率 (2560×1440)
capture_window 的 window_x/y 使用逻辑坐标
mouse_click 的 x/y 使用逻辑坐标

所以：所有工具已经自动处理了 DPI，直接用返回值即可。
```

**注意：** 如果发现点击偏移，先用 `get_screen_info()` 确认返回的分辨率。
如果截图实际像素和返回值不一致，可能是 DPI 问题。

## 验证点击是否准确

```
步骤1: mouse_move(x=target_x, y=target_y)
步骤2: color = get_pixel_color(x=target_x, y=target_y)
步骤3: 检查颜色
  - 如果目标是绿色按钮，期望 RGB ≈ (76, 175, 80)
  - 如果是白色背景，期望 RGB ≈ (255, 255, 255)
  - 如果颜色完全不对，说明坐标有误，需要微调
```
