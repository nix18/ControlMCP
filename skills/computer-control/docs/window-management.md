# 窗口管理参考

## 核心判断

“窗口在前台”不等于“窗口已经进入可操作状态”。

常见异常：
- 窗口被最小化后恢复成小尺寸
- 窗口被其他应用挤压，只显示局部区域
- 窗口停在分屏状态，导致操作区不完整
- 目标窗口虽然聚焦，但并不是你以为的那个项目/应用实例
- `focus_window()` 返回成功，但目标窗口没有真实出现在桌面上

## 托盘窗口判定

如果满足下面这个现象：

```text
focus_window("目标窗口") -> 返回成功
但截图/肉眼验证时，目标窗口仍没有出现在桌面上
```

默认按“系统托盘窗口”处理，而不是继续重复调用 `focus_window()`。

推荐处理顺序：

```text
1. Win+B
2. wait(0.5)
3. Enter 打开托盘/溢出区
4. 必要时方向键定位目标图标
5. Enter 恢复主窗口
6. capture_screen / capture_window 验证
```

原则：
- `focus_window()` 解决的是窗口句柄聚焦，不保证托盘业务态恢复
- “focus 后仍不出现”优先说明入口不在普通窗口层，而在系统托盘层
- 对这类应用，托盘点击/键盘进入比继续重试聚焦更稳定

## 常用快捷键

### Windows

| 目标 | 快捷键 | 说明 |
|------|--------|------|
| 最大化窗口 | `Win + Up` | 目标窗口聚焦后优先使用 |
| 恢复/最小化窗口 | `Win + Down` | 一次恢复普通大小，再按一次通常最小化 |
| 左对齐分屏 | `Win + Left` | 并排对照时用 |
| 右对齐分屏 | `Win + Right` | 并排对照时用 |
| 切换窗口 | `Alt + Tab` | 找到正确应用实例 |
| 显示桌面 | `Win + D` | 快速清场 |

## 标准流程

### 场景 1：窗口可能被最小化

```text
1. find_windows(title_contains="目标标题")
2. focus_window(title="目标标题")
3. key_press(["win", "up"])
4. wait(0.5)
5. capture_window(title="目标标题")
```

### 场景 2：需要把两个窗口并排观察

```text
1. focus_window("窗口A")
2. Win+Left
3. focus_window("窗口B")
4. Win+Right
5. 分别截图确认
```

### 场景 3：窗口标题相似，可能切错实例

```text
1. find_windows(title_contains="关键词")
2. 对照返回的标题选择更精确的 focus_window 参数
3. focus 后截图确认
```

## 实战经验

### 经验 1：`focus_window()` 之后最好立刻做一次窗口修正

尤其是 IDE、浏览器、设计软件这类复杂窗口。

推荐默认动作：

```text
focus_window → Win+Up → wait(0.5)
```

原因：
- 最小化窗口恢复后常常不是最大化状态
- 局部截图容易导致误判

### 经验 2：截图中只看到“半个窗口”时，不要继续业务操作

先修窗口，再谈点击、运行、读取日志。

### 经验 3：当需要保持某个布局时，才使用分屏

平时默认先最大化。分屏是任务驱动的例外，不是默认布局。
