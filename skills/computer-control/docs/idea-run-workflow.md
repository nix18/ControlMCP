# IDEA 启动与日志观察

## 适用场景

适用于这类请求：
- 启动某个 IDEA / PyCharm 运行配置
- 等控制台日志停止更新再汇报
- 检查当前运行项是否正确

## 操作流程

### 1. 进入正确窗口

```text
1. find_windows(title_contains="项目名")
2. focus_window(title="项目名")
3. Win+Up
4. wait(0.5)
5. capture_window 验证
```

## 2. 确认运行配置

先看顶部运行配置名称。

如果不确定当前配置是否正确：

```text
1. Alt+Shift+F10
2. 输入目标配置名
3. Enter 选择
```

不要直接假设顶部缩写一定对应目标配置。

## 3. 启动应用

常用动作：
- 启动当前配置(无运行子窗口或运行子窗口配置非所需启动的配置)：`Shift+F10`
- 启动当前配置(有运行子窗口且运行子窗口配置是所需启动的配置)重启当前配置：`Ctrl+F5`
- 调试模式运行：`Shift+F9`
- 停止当前运行：`Ctrl+F2`
- 打开 Run 面板：`Alt+4`

## 3.1 快捷键不达预期时的兜底资料

JetBrains 系列 IDE 的快捷键经验要带一个前提：
- 不同 IDE 产品可能不同
- 不同版本可能不同
- 不同电脑上的安装路径可能不同
- 用户如果改过 keymap，实际行为也可能不同

因此，快捷键不达预期时按这个顺序处理：

1. 先查本机对应 IDE 的快捷键 PDF 参考卡
2. 再查 JetBrains 官方在线文档
3. 依据当前 IDE、当前版本、当前 keymap 修正动作

本机 IntelliJ IDEA 示例 PDF：

```text
C:\Program Files\JetBrains\IntelliJ IDEA 2025.3.2\help\ReferenceCard.pdf
```

注意：
- 这个路径只是示例，不应写死为唯一位置
- PyCharm、WebStorm、GoLand 等产品名称和安装目录可能不同
- 同一产品不同版本，目录名也会变化

IDEA 官方在线资料示例：

```text
https://www.jetbrains.com.cn/idea/resources/
```

适用原则：
- 快捷键按下去无反应
- 快捷键触发了别的动作
- 点击位置不符合预期，需要回到官方资料确认入口名称和 UI 位置

## 4. 确认控制台面板

启动后，Run 面板可能出现两种状态：

### 状态 A：引导页

特征：
- 面板里是“要运行代码，请执行以下操作”
- 看不到实际控制台输出

这不代表程序没启动，只代表你还没切到实际运行项。

### 状态 B：控制台页

特征：
- 能看到 `java.exe` 命令行、Spring 日志、时间戳或 stdout 输出

## 5. 如果还在引导页

处理方式：
- 检查底部 Run 标签栏是否有正在运行的应用标签
- 切换到对应标签
- 必要时点击“正在运行应用程序”这类入口

原则：
- 不要把“引导页”误判成“没有日志”

## 6. 判断日志是否停止更新

不要只观察一次。

推荐轮询：

```text
1. wait(5-10s)
2. capture_window
3. 记录最后时间戳或最后 2-3 行
4. 再 wait(5-10s)
5. capture_window
6. 再比较一次
7. 如果两轮一致，再进行第三轮确认
```

汇报“日志已停止更新”的建议标准：
- 连续两轮以上无新增输出
- 最后一批日志时间戳不再推进

## 7. 常见误判

### 误判 1：看到 Run 面板就以为已经在看控制台

实际上可能只是引导页。

### 误判 2：只凭一次截图就判断“日志停了”

日志可能只是间歇性输出。

### 误判 3：顶部配置正确，就默认底部运行标签也正确

实际上底部可能切在另一个运行项，或者根本还没切到控制台。

### 误判 4：以为 JetBrains 系列 IDE 的快捷键在所有机器都完全一致

实际上还会受产品、版本、安装目录和用户 keymap 影响。

修正：
- 先查本机 PDF 参考卡
- 再查官方在线文档
